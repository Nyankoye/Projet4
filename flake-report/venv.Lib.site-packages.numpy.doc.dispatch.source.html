1 """.. _dispatch_mechanism:
2  
3 Numpy's dispatch mechanism, introduced in numpy version v1.16 is the
4 recommended approach for writing custom N-dimensional array containers that are
5 compatible with the numpy API and provide custom implementations of numpy
6 functionality. Applications include `dask <http://dask.pydata.org>`_ arrays, an
7 N-dimensional array distributed across multiple nodes, and `cupy
8 <https://docs-cupy.chainer.org/en/stable/>`_ arrays, an N-dimensional array on
9 a GPU.
10  
11 To get a feel for writing custom array containers, we'll begin with a simple
12 example that has rather narrow utility but illustrates the concepts involved.
13  
14 >>> import numpy as np
15 >>> class DiagonalArray:
16 ... def __init__(self, N, value):
17 ... self._N = N
18 ... self._i = value
19 ... def __repr__(self):
20 ... return f"{self.__class__.__name__}(N={self._N}, value={self._i})"
21 ... def __array__(self):
22 ... return self._i * np.eye(self._N)
23 ...
24  
25 Our custom array can be instantiated like:
26  
27 >>> arr = DiagonalArray(5, 1)
28 >>> arr
29 DiagonalArray(N=5, value=1)
30  
31 We can convert to a numpy array using :func:`numpy.array` or
32 :func:`numpy.asarray`, which will call its ``__array__`` method to obtain a
33 standard ``numpy.ndarray``.
34  
35 >>> np.asarray(arr)
36 array([[1., 0., 0., 0., 0.],
37 [0., 1., 0., 0., 0.],
38 [0., 0., 1., 0., 0.],
39 [0., 0., 0., 1., 0.],
40 [0., 0., 0., 0., 1.]])
41  
42 If we operate on ``arr`` with a numpy function, numpy will again use the
43 ``__array__`` interface to convert it to an array and then apply the function
44 in the usual way.
45  
46 >>> np.multiply(arr, 2)
47 array([[2., 0., 0., 0., 0.],
48 [0., 2., 0., 0., 0.],
49 [0., 0., 2., 0., 0.],
50 [0., 0., 0., 2., 0.],
51 [0., 0., 0., 0., 2.]])
52  
53  
54 Notice that the return type is a standard ``numpy.ndarray``.
55  
56 >>> type(arr)
57 numpy.ndarray
58  
59 How can we pass our custom array type through this function? Numpy allows a
60 class to indicate that it would like to handle computations in a custom-defined
61 way through the interfaces ``__array_ufunc__`` and ``__array_function__``. Let's
62 take one at a time, starting with ``_array_ufunc__``. This method covers
63 :ref:`ufuncs`, a class of functions that includes, for example,
64 :func:`numpy.multiply` and :func:`numpy.sin`.
65  
66 The ``__array_ufunc__`` receives:
67  
68 - ``ufunc``, a function like ``numpy.multiply``
69 - ``method``, a string, differentiating between ``numpy.multiply(...)`` and
70 variants like ``numpy.multiply.outer``, ``numpy.multiply.accumulate``, and so
71 on. For the common case, ``numpy.multiply(...)``, ``method == '__call__'``.
72 - ``inputs``, which could be a mixture of different types
73 - ``kwargs``, keyword arguments passed to the function
74  
75 For this example we will only handle the method ``__call__``.
76  
77 >>> from numbers import Number
78 >>> class DiagonalArray:
79 ... def __init__(self, N, value):
80 ... self._N = N
81 ... self._i = value
82 ... def __repr__(self):
83 ... return f"{self.__class__.__name__}(N={self._N}, value={self._i})"
84 ... def __array__(self):
85 ... return self._i * np.eye(self._N)
86 ... def __array_ufunc__(self, ufunc, method, *inputs, **kwargs):
87 ... if method == '__call__':
88 ... N = None
89 ... scalars = []
90 ... for input in inputs:
91 ... if isinstance(input, Number):
92 ... scalars.append(input)
93 ... elif isinstance(input, self.__class__):
94 ... scalars.append(input._i)
95 ... if N is not None:
96 ... if N != self._N:
97 ... raise TypeError("inconsistent sizes")
98 ... else:
99 ... N = self._N
100 ... else:
101 ... return NotImplemented
102 ... return self.__class__(N, ufunc(*scalars, **kwargs))
103 ... else:
104 ... return NotImplemented
105 ...
106  
107 Now our custom array type passes through numpy functions.
108  
109 >>> arr = DiagonalArray(5, 1)
110 >>> np.multiply(arr, 3)
111 DiagonalArray(N=5, value=3)
112 >>> np.add(arr, 3)
113 DiagonalArray(N=5, value=4)
114 >>> np.sin(arr)
115 DiagonalArray(N=5, value=0.8414709848078965)
116  
117 At this point ``arr + 3`` does not work.
118  
119 >>> arr + 3
120 TypeError: unsupported operand type(s) for *: 'DiagonalArray' and 'int'
121  
122 To support it, we need to define the Python interfaces ``__add__``, ``__lt__``,
123 and so on to dispatch to the corresponding ufunc. We can achieve this
124 conveniently by inheriting from the mixin
125 :class:`~numpy.lib.mixins.NDArrayOperatorsMixin`.
126  
127 >>> import numpy.lib.mixins
128 >>> class DiagonalArray(numpy.lib.mixins.NDArrayOperatorsMixin):
129 ... def __init__(self, N, value):
130 ... self._N = N
131 ... self._i = value
132 ... def __repr__(self):
133 ... return f"{self.__class__.__name__}(N={self._N}, value={self._i})"
134 ... def __array__(self):
135 ... return self._i * np.eye(self._N)
136 ... def __array_ufunc__(self, ufunc, method, *inputs, **kwargs):
137 ... if method == '__call__':
138 ... N = None
139 ... scalars = []
140 ... for input in inputs:
141 ... if isinstance(input, Number):
142 ... scalars.append(input)
143 ... elif isinstance(input, self.__class__):
144 ... scalars.append(input._i)
145 ... if N is not None:
146 ... if N != self._N:
147 ... raise TypeError("inconsistent sizes")
148 ... else:
149 ... N = self._N
150 ... else:
151 ... return NotImplemented
152 ... return self.__class__(N, ufunc(*scalars, **kwargs))
153 ... else:
154 ... return NotImplemented
155 ...
156  
157 >>> arr = DiagonalArray(5, 1)
158 >>> arr + 3
159 DiagonalArray(N=5, value=4)
160 >>> arr > 0
161 DiagonalArray(N=5, value=True)
162  
163 Now let's tackle ``__array_function__``. We'll create dict that maps numpy
164 functions to our custom variants.
165  
166 >>> HANDLED_FUNCTIONS = {}
167 >>> class DiagonalArray(numpy.lib.mixins.NDArrayOperatorsMixin):
168 ... def __init__(self, N, value):
169 ... self._N = N
170 ... self._i = value
171 ... def __repr__(self):
172 ... return f"{self.__class__.__name__}(N={self._N}, value={self._i})"
173 ... def __array__(self):
174 ... return self._i * np.eye(self._N)
175 ... def __array_ufunc__(self, ufunc, method, *inputs, **kwargs):
176 ... if method == '__call__':
177 ... N = None
178 ... scalars = []
179 ... for input in inputs:
180 ... # In this case we accept only scalar numbers or DiagonalArrays.
181 ... if isinstance(input, Number):
182 ... scalars.append(input)
183 ... elif isinstance(input, self.__class__):
184 ... scalars.append(input._i)
185 ... if N is not None:
186 ... if N != self._N:
187 ... raise TypeError("inconsistent sizes")
188 ... else:
189 ... N = self._N
190 ... else:
191 ... return NotImplemented
192 ... return self.__class__(N, ufunc(*scalars, **kwargs))
193 ... else:
194 ... return NotImplemented
195 ... def __array_function__(self, func, types, args, kwargs):
196 ... if func not in HANDLED_FUNCTIONS:
197 ... return NotImplemented
198 ... # Note: this allows subclasses that don't override
199 ... # __array_function__ to handle DiagonalArray objects.
200 ... if not all(issubclass(t, self.__class__) for t in types):
201 ... return NotImplemented
202 ... return HANDLED_FUNCTIONS[func](*args, **kwargs)
203 ...
204  
205 A convenient pattern is to define a decorator ``implements`` that can be used
206 to add functions to ``HANDLED_FUNCTIONS``.
207  
208 >>> def implements(np_function):
209 ... "Register an __array_function__ implementation for DiagonalArray objects."
210 ... def decorator(func):
211 ... HANDLED_FUNCTIONS[np_function] = func
212 ... return func
213 ... return decorator
214 ...
215  
216 Now we write implementations of numpy functions for ``DiagonalArray``.
217 For completeness, to support the usage ``arr.sum()`` add a method ``sum`` that
218 calls ``numpy.sum(self)``, and the same for ``mean``.
219  
220 >>> @implements(np.sum)
221 ... def sum(arr):
222 ... "Implementation of np.sum for DiagonalArray objects"
223 ... return arr._i * arr._N
224 ...
225 >>> @implements(np.mean)
226 ... def mean(arr):
227 ... "Implementation of np.mean for DiagonalArray objects"
228 ... return arr._i / arr._N
229 ...
230 >>> arr = DiagonalArray(5, 1)
231 >>> np.sum(arr)
232 5
233 >>> np.mean(arr)
234 0.2
235  
236 If the user tries to use any numpy functions not included in
237 ``HANDLED_FUNCTIONS``, a ``TypeError`` will be raised by numpy, indicating that
238 this operation is not supported. For example, concatenating two
239 ``DiagonalArrays`` does not produce another diagonal array, so it is not
240 supported.
241  
242 >>> np.concatenate([arr, arr])
  • E501 Line too long (137 > 119 characters)
243 TypeError: no implementation found for 'numpy.concatenate' on types that implement __array_function__: [<class '__main__.DiagonalArray'>]
244  
245 Additionally, our implementations of ``sum`` and ``mean`` do not accept the
246 optional arguments that numpy's implementation does.
247  
248 >>> np.sum(arr, axis=0)
249 TypeError: sum() got an unexpected keyword argument 'axis'
250  
251 The user always has the option of converting to a normal ``numpy.ndarray`` with
252 :func:`numpy.asarray` and using standard numpy from there.
253  
254 >>> np.concatenate([np.asarray(arr), np.asarray(arr)])
255 array([[1., 0., 0., 0., 0.],
256 [0., 1., 0., 0., 0.],
257 [0., 0., 1., 0., 0.],
258 [0., 0., 0., 1., 0.],
259 [0., 0., 0., 0., 1.],
260 [1., 0., 0., 0., 0.],
261 [0., 1., 0., 0., 0.],
262 [0., 0., 1., 0., 0.],
263 [0., 0., 0., 1., 0.],
264 [0., 0., 0., 0., 1.]])
265  
266 Refer to the `dask source code <https://github.com/dask/dask>`_ and
267 `cupy source code <https://github.com/cupy/cupy>`_ for more fully-worked
268 examples of custom array containers.
