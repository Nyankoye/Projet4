8 along an axis
9 Axes are defined for arrays with more than one dimension. A
10 2-dimensional array has two corresponding axes: the first running
11 vertically downwards across rows (axis 0), and the second running
12 horizontally across columns (axis 1).
13  
14 Many operations can take place along one of these axes. For example,
15 we can sum each row of an array, in which case we operate along
16 columns, or axis 1::
17  
18 >>> x = np.arange(12).reshape((3,4))
19  
20 >>> x
21 array([[ 0, 1, 2, 3],
22 [ 4, 5, 6, 7],
23 [ 8, 9, 10, 11]])
24  
25 >>> x.sum(axis=1)
26 array([ 6, 22, 38])
27  
28 array
29 A homogeneous container of numerical elements. Each element in the
30 array occupies a fixed amount of memory (hence homogeneous), and
31 can be a numerical element of a single type (such as float, int
32 or complex) or a combination (such as ``(float, int, float)``). Each
33 array has an associated data-type (or ``dtype``), which describes
34 the numerical type of its elements::
35  
36 >>> x = np.array([1, 2, 3], float)
37  
38 >>> x
39 array([ 1., 2., 3.])
40  
41 >>> x.dtype # floating point number, 64 bits of memory per element
42 dtype('float64')
43  
44  
45 # More complicated data type: each array element is a combination of
46 # and integer and a floating point number
47 >>> np.array([(1, 2.0), (3, 4.0)], dtype=[('x', int), ('y', float)])
48 array([(1, 2.0), (3, 4.0)],
49 dtype=[('x', '<i4'), ('y', '<f8')])
50  
51 Fast element-wise operations, called a :term:`ufunc`, operate on arrays.

53 array_like

65 big-endian
66 When storing a multi-byte value in memory as a sequence of bytes, the
67 sequence addresses/sends/stores the most significant byte first (lowest
68 address) and the least significant byte last (highest address). Common in
69 micro-processors and used for transmission of data over network protocols.

93 C order

96 column-major
97 A way to represent items in a N-dimensional array in the 1-dimensional
98 computer memory. In column-major order, the leftmost index "varies the
99 fastest": for example the array::
100  
101 [[1, 2, 3],
102 [4, 5, 6]]
103  
104 is represented in the column-major order as::
105  
106 [1, 4, 2, 5, 3, 6]
107  
108 Column-major order is also known as the Fortran order, as the Fortran
109 programming language uses it.

111 decorator
112 An operator that transforms a function. For example, a ``log``
113 decorator may be defined to print debugging information upon
114 function execution::
115  
116 >>> def log(f):
117 ... def new_logging_func(*args, **kwargs):
118 ... print("Logging call with parameters:", args, kwargs)
119 ... return f(*args, **kwargs)
120 ...
121 ... return new_logging_func
122  
123 Now, when we define a function, we can "decorate" it using ``log``::
124  
125 >>> @log
126 ... def add(a, b):
127 ... return a + b
128  
129 Calling ``add`` then yields:
130  
131 >>> add(1, 2)
132 Logging call with parameters: (1, 2) {}

135 dictionary
136 Resembling a language dictionary, which provides a mapping between
137 words and descriptions thereof, a Python dictionary is a mapping
138 between two objects::
139  
140 >>> x = {1: 'one', 'two': [1, 2]}
141  
142 Here, `x` is a dictionary mapping keys to values, in this case
143 the integer 1 to the string "one", and the string "two" to
144 the list ``[1, 2]``. The values may be accessed using their
145 corresponding keys::
146  
147 >>> x[1]
148 'one'
149  
150 >>> x['two']
151 [1, 2]
152  
153 Note that dictionaries are not stored in any specific order. Also,
154 most mutable (see *immutable* below) objects, such as lists, may not
155 be used as keys.
156  
157 For more information on dictionaries, read the
158 `Python tutorial <https://docs.python.org/tutorial/>`_.

161 In a :term:`structured data type`, each sub-type is called a `field`.
162 The `field` has a name (a string), a type (any valid dtype), and
163 an optional `title`. See :ref:`arrays.dtypes`

  • W291 Trailing whitespace
173 Describes a block of memory comprised of blocks, each block comprised of
174 items and of the same size, and blocks are interpreted in exactly the

179 An object that cannot be modified after execution is called

182 instance
183 A class definition gives the blueprint for constructing an object::
184  
185 >>> class House:
186 ... wall_colour = 'white'
187  
188 Yet, we have to *build* a house before it exists::
189  
190 >>> h = House() # build a house
191  
192 Now, ``h`` is called a ``House`` instance. An instance is therefore
193 a specific realisation of a class.

195 iterable
196 A sequence that allows "walking" (iterating) over items, typically
197 using a loop such as::
198  
199 >>> x = [1, 2, 3]
200 >>> [item**2 for item in x]
201 [1, 4, 9]
202  
203 It is often used in combination with ``enumerate``::
204 >>> keys = ['a','b','c']
205 >>> for n, k in enumerate(keys):
206 ... print("Key %d: %s" % (n, k))
207 ...
208 Key 0: a
209 Key 1: b
210 Key 2: c

215 list
216 A Python container that can hold any number of objects or items.
217 The items do not have to be of the same type, and can even be
218 lists themselves::
219  
220 >>> x = [2, 2.0, "two", [2, 2.0]]
221  
222 The list `x` contains 4 items, each which can be accessed individually::
223  
224 >>> x[2] # the string 'two'
225 'two'
226  
227 >>> x[3] # a list, containing an integer 2 and a float 2.0
228 [2, 2.0]
229  
230 It is also possible to select more than one item at a time,
231 using *slicing*::
232  
233 >>> x[0:2] # or, equivalently, x[:2]
234 [2, 2.0]
235  
236 In code, arrays are often conveniently expressed as nested lists::
237  
238  
239 >>> np.array([[1, 2], [3, 4]])
240 array([[1, 2],
241 [3, 4]])
242  
243 For more information, read the section on lists in the `Python
244 tutorial <https://docs.python.org/tutorial/>`_. For a mapping
245 type (key-value), see *dictionary*.

248 When storing a multi-byte value in memory as a sequence of bytes, the
249 sequence addresses/sends/stores the least significant byte first (lowest
250 address) and the most significant byte last (highest address). Common in
251 x86 processors.

253 mask
254 A boolean array, used to select only certain elements for an operation::
255  
256 >>> x = np.arange(5)
257 >>> x
258 array([0, 1, 2, 3, 4])
259  
260 >>> mask = (x > 2)
261 >>> mask
262 array([False, False, False, True, True])
263  
264 >>> x[mask] = -1
265 >>> x
266 array([ 0, 1, 2, -1, -1])

268 masked array
269 Array that suppressed values indicated by a mask::
270  
271 >>> x = np.ma.masked_array([np.nan, 2, np.nan], [True, False, True])
272 >>> x
273 masked_array(data = [-- 2.0 --],
274 mask = [ True False True],
275 fill_value = 1e+20)
276  
277 >>> x + [1, 2, 3]
278 masked_array(data = [-- 4.0 --],
279 mask = [ True False True],
280 fill_value = 1e+20)
281  
282  
283 Masked arrays are often used when operating on arrays containing
284 missing or invalid entries.

320 row-major
321 A way to represent items in a N-dimensional array in the 1-dimensional
322 computer memory. In row-major order, the rightmost index "varies
323 the fastest": for example the array::

325 [[1, 2, 3],

335 self
336 Often seen in method signatures, ``self`` refers to the instance
337 of the associated class. For example:
338  
339 >>> class Paintbrush:

341 ...
342 ... def paint(self):

350 slice
351 Used to select only certain elements from a sequence:
352  
353 >>> x = range(5)
354 >>> x
355 [0, 1, 2, 3, 4]
356  
357 >>> x[1:3] # slice from 1 to 3 (excluding 3 itself)
358 [1, 2]
359  
360 >>> x[1:5:2] # slice from 1 to 5, but skipping every second element
361 [1, 3]
362  
363 >>> x[::-1] # slice a sequence in reverse
364 [4, 3, 2, 1, 0]
365  
366 Arrays may have more than one dimension, each which can be sliced
367 individually:
368  
369 >>> x = np.array([[1, 2], [3, 4]])
370 >>> x
371 array([[1, 2],
372 [3, 4]])
373  
374 >>> x[:, 1]
375 array([2, 4])

397 tuple
398 A sequence that may contain a variable number of types of any
399 kind. A tuple is immutable, i.e., once constructed it cannot be
400 changed. Similar to a list, it can be indexed and sliced::
401  
402 >>> x = (1, 'one', [1, 2])
403 >>> x
404 (1, 'one', [1, 2])
405  
406 >>> x[0]
407 1
408  
409 >>> x[:2]
410 (1, 'one')
411  
412 A useful concept is "tuple unpacking", which allows variables to
413 be assigned to the contents of a tuple::
414  
415 >>> x, y = (1, 2)
416 >>> x, y = 1, 2
417  
418 This is often used when a function returns multiple values:
419  
420 >>> def return_many():
421 ... return 1, 'alpha', None
422  
423 >>> a, b, c = return_many()
424 >>> a, b, c
425 (1, 'alpha', None)
426  
427 >>> a
428 1
429 >>> b
430 'alpha'

432 ufunc
433 Universal function. A fast element-wise, :term:`vectorized
434 <vectorization>` array operation. Examples include ``add``, ``sin`` and
435 ``logical_or``.
436  
437 vectorization
438 Optimizing a looping block by specialized code. In a traditional sense,
439 vectorization performs the same operation on multiple elements with
440 fixed strides between them via specialized hardware. Compilers know how
441 to take advantage of well-constructed loops to implement such
442 optimizations. NumPy uses :ref:`vectorization <whatis-vectorization>`
443 to mean any optimization via specialized code performing the same
444 operations on multiple elements, typically achieving speedups by
445 avoiding some of the overhead in looking up and converting the elements.
446  
447 view
448 An array that does not own its data, but refers to another array's
449 data instead. For example, we may create a view that only shows
450 every second element of another array::
451  
452 >>> x = np.arange(5)
453 >>> x
454 array([0, 1, 2, 3, 4])
455  
456 >>> y = x[::2]
457 >>> y
458 array([0, 2, 4])
459  
460 >>> x[0] = 3 # changing x changes y as well, since y is a view on x
461 >>> y
462 array([3, 2, 4])

464 wrapper
465 Python is a high-level (highly abstracted, or English-like) language.
466 This abstraction comes at a price in execution speed, and sometimes
467 it becomes necessary to use lower level languages to do fast
468 computations. A wrapper is code that provides a bridge between
469 high and the low level languages, allowing, e.g., Python to execute
470 code written in C or Fortran.
471  
472 Examples include ctypes, SWIG and Cython (which wraps C and C++)
473 and f2py (which wraps Fortran).
