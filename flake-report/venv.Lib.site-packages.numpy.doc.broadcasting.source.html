2 ========================
3 Broadcasting over arrays
4 ========================
5  
6 .. note::
7 See `this article
8 <https://numpy.org/devdocs/user/theory.broadcasting.html>`_
9 for illustrations of broadcasting concepts.
10  
11  
12 The term broadcasting describes how numpy treats arrays with different
13 shapes during arithmetic operations. Subject to certain constraints,
14 the smaller array is "broadcast" across the larger array so that they
15 have compatible shapes. Broadcasting provides a means of vectorizing
16 array operations so that looping occurs in C instead of Python. It does
17 this without making needless copies of data and usually leads to
18 efficient algorithm implementations. There are, however, cases where
19 broadcasting is a bad idea because it leads to inefficient use of memory
20 that slows computation.
21  
22 NumPy operations are usually done on pairs of arrays on an
23 element-by-element basis. In the simplest case, the two arrays must
24 have exactly the same shape, as in the following example:
25  
26 >>> a = np.array([1.0, 2.0, 3.0])
27 >>> b = np.array([2.0, 2.0, 2.0])
28 >>> a * b
29 array([ 2., 4., 6.])
30  
31 NumPy's broadcasting rule relaxes this constraint when the arrays'
32 shapes meet certain constraints. The simplest broadcasting example occurs
33 when an array and a scalar value are combined in an operation:
34  
35 >>> a = np.array([1.0, 2.0, 3.0])
36 >>> b = 2.0
37 >>> a * b
38 array([ 2., 4., 6.])
39  
40 The result is equivalent to the previous example where ``b`` was an array.
41 We can think of the scalar ``b`` being *stretched* during the arithmetic
42 operation into an array with the same shape as ``a``. The new elements in
43 ``b`` are simply copies of the original scalar. The stretching analogy is
44 only conceptual. NumPy is smart enough to use the original scalar value
45 without actually making copies so that broadcasting operations are as
46 memory and computationally efficient as possible.
47  
48 The code in the second example is more efficient than that in the first
49 because broadcasting moves less memory around during the multiplication
50 (``b`` is a scalar rather than an array).
51  
52 General Broadcasting Rules
53 ==========================
54 When operating on two arrays, NumPy compares their shapes element-wise.
55 It starts with the trailing dimensions and works its way forward. Two
56 dimensions are compatible when
57  
58 1) they are equal, or
59 2) one of them is 1
60  
61 If these conditions are not met, a
  • W291 Trailing whitespace
62 ``ValueError: operands could not be broadcast together`` exception is
  • W291 Trailing whitespace
63 thrown, indicating that the arrays have incompatible shapes. The size of
64 the resulting array is the size that is not 1 along each axis of the inputs.
65  
66 Arrays do not need to have the same *number* of dimensions. For example,
67 if you have a ``256x256x3`` array of RGB values, and you want to scale
68 each color in the image by a different value, you can multiply the image
69 by a one-dimensional array with 3 values. Lining up the sizes of the
70 trailing axes of these arrays according to the broadcast rules, shows that
71 they are compatible::
72  
73 Image (3d array): 256 x 256 x 3
74 Scale (1d array): 3
75 Result (3d array): 256 x 256 x 3
76  
77 When either of the dimensions compared is one, the other is
78 used. In other words, dimensions with size 1 are stretched or "copied"
79 to match the other.
80  
81 In the following example, both the ``A`` and ``B`` arrays have axes with
82 length one that are expanded to a larger size during the broadcast
83 operation::
84  
85 A (4d array): 8 x 1 x 6 x 1
86 B (3d array): 7 x 1 x 5
87 Result (4d array): 8 x 7 x 6 x 5
88  
89 Here are some more examples::
90  
91 A (2d array): 5 x 4
92 B (1d array): 1
93 Result (2d array): 5 x 4
94  
95 A (2d array): 5 x 4
96 B (1d array): 4
97 Result (2d array): 5 x 4
98  
99 A (3d array): 15 x 3 x 5
100 B (3d array): 15 x 1 x 5
101 Result (3d array): 15 x 3 x 5
102  
103 A (3d array): 15 x 3 x 5
104 B (2d array): 3 x 5
105 Result (3d array): 15 x 3 x 5
106  
107 A (3d array): 15 x 3 x 5
108 B (2d array): 3 x 1
109 Result (3d array): 15 x 3 x 5
110  
111 Here are examples of shapes that do not broadcast::
112  
113 A (1d array): 3
114 B (1d array): 4 # trailing dimensions do not match
115  
116 A (2d array): 2 x 1
117 B (3d array): 8 x 4 x 3 # second from last dimensions mismatched
118  
119 An example of broadcasting in practice::
120  
121 >>> x = np.arange(4)
122 >>> xx = x.reshape(4,1)
123 >>> y = np.ones(5)
124 >>> z = np.ones((3,4))
125  
126 >>> x.shape
127 (4,)
128  
129 >>> y.shape
130 (5,)
131  
132 >>> x + y
133 ValueError: operands could not be broadcast together with shapes (4,) (5,)
134  
135 >>> xx.shape
136 (4, 1)
137  
138 >>> y.shape
139 (5,)
140  
141 >>> (xx + y).shape
142 (4, 5)
143  
144 >>> xx + y
145 array([[ 1., 1., 1., 1., 1.],
146 [ 2., 2., 2., 2., 2.],
147 [ 3., 3., 3., 3., 3.],
148 [ 4., 4., 4., 4., 4.]])
149  
150 >>> x.shape
151 (4,)
152  
153 >>> z.shape
154 (3, 4)
155  
156 >>> (x + z).shape
157 (3, 4)
158  
159 >>> x + z
160 array([[ 1., 2., 3., 4.],
161 [ 1., 2., 3., 4.],
162 [ 1., 2., 3., 4.]])
163  
164 Broadcasting provides a convenient way of taking the outer product (or
165 any other outer operation) of two arrays. The following example shows an
166 outer addition operation of two 1-d arrays::
167  
168 >>> a = np.array([0.0, 10.0, 20.0, 30.0])
169 >>> b = np.array([1.0, 2.0, 3.0])
170 >>> a[:, np.newaxis] + b
171 array([[ 1., 2., 3.],
172 [ 11., 12., 13.],
173 [ 21., 22., 23.],
174 [ 31., 32., 33.]])
175  
176 Here the ``newaxis`` index operator inserts a new axis into ``a``,
177 making it a two-dimensional ``4x1`` array. Combining the ``4x1`` array
178 with ``b``, which has shape ``(3,)``, yields a ``4x3`` array.
